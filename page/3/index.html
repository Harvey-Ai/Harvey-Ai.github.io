查找树总结

2014-10-06

查找树的种类很多，最基本的是二分查找树，然后有为解决平衡性的红黑树和AVL树，然后有为了减小树高的B树和B+树。这些查找树的目的都是为了查找，将数据按树的性状进行组织，便与以后的查找。

二分查找树

二分查找树是最基本的，如果没有特殊需求，二分查找树能应付大部分的查找问题。二分查找树将数据组织成二叉树形式，左儿子所在子树中节点都小于当前节点值，而右儿子所在子树中节点都大于当前节点值，所以该树的中根遍历会是一个有序序列。

线段数总结

2014-10-06

在日常数据处理中，经常会遇到对一段数据的修改与查询，如果朴素地对待这个问题，时间复杂度上会承受不了，这里介绍两种处理这类问题的数据结构——线段树、树状数组。

线段树

虽然名字挺起来很高端，但是起始线段树仅仅是一个普通的二叉树，只不过树节点被特化为表示一段数值，而不是一个值，并没有特别可怕的地方。
线段树提供修改和查询一段数据的操作，删除和添加可通过特殊标记达到。那么树上的节点到底是如何表示线段的呢？线段树对线段的表示是这样的，假设当前节点表示的线段范围是[a, b]，那么左子节点表示范围为 [a, (a + b) / 2]，右子节点表示范围为[(a + b) / 2 + 1, b]，这样一直递归到叶子节点，叶子节点仅表示一个点的数值。如下图所示，表示一个[1, 10]的线段树。

并查集总结

2014-10-05

并查集，从字面上看就能看出这种数据结构的功能，用于不相交集合的合并和查询。所以总结一下，使用并查集要记住三点：

  • 并查集只能表示不相交的集合。
  • 并查集只提供合并和查询操作，没有拆分和删除操作，这意味这表示的集合会变得越来越大，所以并查集的主要在于一个”并”字，是用于合并的集合的。

2014-09-17

概述

动态规划是ICPC中使用非常频繁的一类编程技巧。所谓动态规划（DP）也就是以状态为基础的多段决策，有对单个状态规划到最终求得全局最优解。动态规划有三个必备条件，那就是：状态、状态转移、状态的无后效性。本质上，动态规划是搜索的一种人工优化，能用动态规划解决的问题自然也能通过搜索进行暴力解决，不过当一些问题的状态转移的层次不是非常明显的时候，记忆化搜索也是非常好的一种取代方法。
