一道贪心题

2009-11-16 | Algorithm

John大叔去伐木，留下N (2 ≤ N ≤ 100,000)只牛在吃草。当他回来的时候，发现恐怖的事情发生了，那些牛正在啃John大叔漂亮的花。John大叔决定马上采取行动，把牛送回牛栏。

每只牛i距离其所属的牛栏需要Ti分钟(1 ≤ Ti ≤ 2,000,000)，它在等待被送回去之前，每分钟会破坏Di (1 ≤ Di ≤ 100)朵花。无论John大叔如何努力，他每次只能够送1只牛回牛栏。将第i只牛送回牛栏需要2 × Ti的时间（Ti时间送去牛栏，Ti时间返回花坛）。John大叔现在在花坛里，将牛们送回牛栏。

请编写一个程序，安排John大叔将牛送回牛栏的顺序，使损失的花朵最少。

输入

第一行为N。

之后N行，每行两个数字，分别表示Ti和Di，描述每只牛被送回所需要的分钟和每分钟破坏的花朵数。

输出

输出仅有一行，表示最少破坏的花朵数。

Sample Input

6
3 1
2 5
2 3
3 2
4 1
1 6

Sample Output
86

题目是从别人的blog上转过来的，方法是贪心，其他方法都不好用，这里就不说了。

这里只想说一下贪心的证明过程，存在两种情况，1.DI>DJ && TI>TJ     2.DI>DJ && TI<TJ
对于第二种情况显然是先把牛I拖回去。
那么第一种情况呢，如果先拖牛I，那么这段时间内，除了其他牛吃的，牛J多出了TI的时间，先拖牛J呢，那么这段时间，除了其他牛吃的，牛I多出了TJ的时间 。好，那么只要判断DITJ和DJTI的大小就可以确定那只牛被先拖回去，转化一下，即比较DI/TI和DJ/TJ的大小。

假设是贪心策略是DI/TI，回到第二种情况，显然适用。那么，策略成立。

这种含有两种元素相互制约，往往是通过“效率/耗费”值来作为贪心值。
