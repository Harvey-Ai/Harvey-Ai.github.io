HTML基本语法总结

2014-09-23 | Language

之前有看过W3C的HTML教程，不是HTML5的教程，是入门级的东西，就在这里记一笔吧。

概况

HTML是标记性语言，语言能力不强，但是对于网页却很合适。HTML通过对标签进行解析，识别网页描述内容。

  • HTML是我记忆中唯一不区分大小写的语言。
  • 标签用<>标示，注释用<!–This is a comment–>进行标示。
  • HTML会忽略连续长度大于一的空格，所有空格被视为一个空格。
  • 属性需要用双引号进行引用，属性内容本身有双引号时，使用单引号进行引用。

开头

  • 文件类型声明：起始部分声明文件类型，HTML5仅用<！DOCTYPE html>就可以，低版本声明文件类型较繁琐。

  • 头声明：用<head>标签进行包含，包括网页信息、访问控制等元信息 (<meta>)和引用文件信息（ <link>引用CSS, <script>引用其他教本）。例如：

    • <meta http-equiv=”charset” content=”utf-8” />，说明网页使用utf-8字符集。
    • <link rel=”stylesheet” type=”text/css” href=”./music.css” />，引用music.css格式文件。
    • <script src=’../lib/jquery-2.1.0.min.js’></script>，引用jquery的javascript文件。

正文

正文用<body>标签包含，包括了html整个显示部分和响应逻辑，更细化的处理可能还包括页眉和页脚。现代的HTML一般运用DIV+CSS对网页进行布局，用JavaScript处理交互。

常用标签

标签说明了内容的是什么，而具体表现会在CSS中处理。下面列举一些常用的标签。

  • 基本格式：

    • <hi>：标题，对不同大小的标题分别有<h1>，<h2>到<h5>。
    • <p>：段落。
    • <a href = “XXX”, target=”XX”>：链接。
    • <img src=”XXX”>：图片。
    • <hr />：水平分割线。
    • <br />：换行。
    • <a href=”#anchor1”> 链接 <a name=”anchor1”>Useful Tips Section</a>： 锚链接。
    • <table>：表格，<th>表格头，<caption>标题，<tr>（行），<td>（列）

  • 文本样式

    • <b>：粗体， <strong>：定义加重语气，<i>：斜体。
    • <small>：小号字体，<big>：大号字体。
    • <sub>：下标，<sup>：上标，<del>：删除字。
    • <code>：代码，<pre>预定义格式。

  • 表单

    • <input>：输入标签，属性type指定输入区样式。Eg: <input type=”reset” value=”重置”>，指定一个重置按钮。
    • <form>：表单，属性action指定表单提交地址，method指定提交方式。Eg: <form name=”input” action=”html_form_action.asp” method=”get”> 指定了表单提交地址和使用get方法提交。

常用属性

HTML里面除了每个标签自带的属性外，另外可以自定义属性，不过有一些属性比较特殊。

  • class：指定元素类别，在CSS和javascript中可以通过”.+class”的形式选择元素。Eg: <p class = “classA”>，通过”.classA”对该段落进行选择。
  • id：指定单个元素，在CSS和javascript中可以通过”#+class”的形式选择元素。Eg: <p class = “myP”>，通过”#myP”对该段落进行选择。
  • style：指定元素的CSS样式，CSS有两种指定样式的方式，其中一种叫内部样式，通过style进行指定。Eg: <p style = “background: blue;”>，指定段落的背景颜色为蓝色。

布局处理

现代HTML的处理是将表现层抽象了出来，具体表现样式是交给CSS处理的，这样在进行布局的基本思路就是：先确定格式，在对不同的标签进行具体布局。

按照对空间的需求可以将元素分为：块级元素，行内元素。顾名思义，块级元素占有网页中一块区域（如<div>），行内元素仅占有网页中一行（如<span>）。

响应处理

HTML以事件模型对交互进行处理，当事件发生时，可以指定回调函数来对事件进行处理，另外一点需要注意的是，各个浏览器对事件的处理方式不太一样，有冒泡模型和捕捉模型，区别在于事件的发生顺序，前者是自底而上，后者是自上而下。

  • onload：指定文档载入。
  • onfocus：元素获得焦点。
  • onkeypress，onkeydown，onkeyup ： 键盘事件。
  • onclick，ondbclick：鼠标单双击事件。
  • onchange，onsubmit：元素发生改变事件。
  • onmouseout，onmouseover：鼠标移入、移出事件。
  • onmousemove，onmousedown，onmouseup：鼠标移动、按下事件。
    事件的处理函数一般是通过javascript定义，在HTML中使用<script type=”text/javascript”>定义javascript的相关响应函数，在元素的的事件回调中指定响应函数。

HTML是前端技术的骨架，先搭好了一个HTML就构成了前端的骨干，在通过CSS的样式修饰和javascript的响应，就能构成一个整体的前端部分了。
