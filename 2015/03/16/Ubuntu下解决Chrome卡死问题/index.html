Ubuntu下解决Chrome卡死问题

2015-03-16 | Engineer

去年因为VirtualBox与Fedora版本总是不兼容，一怒之下删除了使用了多年的Fedora，安装上Ubuntu14.04 TLS。Ubuntu用起来非常舒服，比Fedora流畅很多。

不过，用过一阵子之后发现当Chrome打开多个网页的时候，偶尔会出现系统疯狂读写硬盘，卡死的情况。上网找了一下，有人说是Chrome的自带Flash造成的原因，然后就没太在意，毕竟发生的频率不高。最近，在做Android开发，要开模拟器，机器动不动就卡死，上网找了一圈，有人说是Swap的问题。

Top一下，发现机器的Swap是0，Ubuntu的默认安装没有添加Swap空间，也是醉了，怪不得出现疯狂读写硬盘的情况，问题应该就在这里。然后按以下步骤添加Swap空间。

#cd /
1） 添加Swap分区， 4G
#dd if=/dev/zero of=swap bs=1M count=4096
2） 格式化
#mkswap swap
3） 打开Swap空间
#swapon swap
4） 编辑/etc/fstab，添加下面的一行，增加分区启动项。(这里不能用UUID，因为并不是一个分区)
/swap   /swap   swap    sw 0    0

添加之后无需重启，运行流畅，再没出现Chrome卡死的情况。
