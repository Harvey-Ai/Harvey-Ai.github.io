Srm595 DVI2

Dec 18, 2013 | Algorithm

1. (DIV2)1000分题，题目大意：给定A, B, C，问存在多少对(X, Y, Z)满足条件0 =< X <= A，0 =< Y <= B，0 =< Z <= C 并且 X ^ Y <= Z。其中A, B, C不大于10e9。

解题思路：首先枚举X, Y是不现实的，n^2的复杂度太大了，于是需要找这个问题的重叠性。可以看到，数字的前缀是可能重复的，如果能知道所有N+1位前缀的答案个数，就能递推出某个N位前缀的答案数，分四种情况计算。例如所有前缀为123X的X，前缀为234X的Y的答案对数知道，那么以123和234为前缀X，Y的答案对数可以分四种情况计算（列举2种）。

  • 如果123和A的前三位相同，234和B的前三位也相同，那么f(123, 234) = sum(f(123x, 234y))，其中x <= A的第四位数，y <= B的第四位数。

  • 如果123和A的前三位相同，234和B的前三位不同，那么f(123, 234) = sum(f(123x, 234y))，其中x <= A的第四位数，0 =< y <= 9。
    通过枚举X，Y的位，可以降低复杂度，这样做其实已经可以了，毕竟状态数也不是非常多，不过还可以进一步压缩状态的个数。回头再看一下状态递推的情况，如果前N位不同后面一个数可以都是任意一个数，所以可以将前N位不同的情况聚集起来存储到一个状态中，即“前N位不同”，所以状态就分为两种，“相同”和“不同”，所以状态转移可以写成

  • f(与A前N位相同, 与B前N位相同) = sum(f(A前N位 + 小于A第（N + 1）位数, B前N位 + 小于B第（N + 1）位数))。

  • f(与A前N位相同, 与B前N位不同) = sum(f(A前N位 + 小于A第（N + 1）位数, B前N位 + 任意))。
    2. (DIV1) 500分题，题目大意：给一个字符串，里面可能包含若干字符‘G’，如”RRRRGBGGGB”。想在可恶的小象想找出任意两个不相交的子字符串，使得这两个子字符串包含知道N个连续的‘G’，问小象可以找到多少个这样的字符串对？（字符串长度不超过2500）

解题思路： 首先可以暴力枚举两个子字符串，复杂度位N^4，显然是超过时间限制的。但是枚举一个子字符串是可以的，如果能直接知道与这个字符串拼接满足答案的字符串数的个数就可以解出这题了。最直接的想法就是建立子字符串索引，然后枚举左边的字符串a，分3种情况（k < N)，

  • 如果字符串a中包含N个连续的‘G’，那么字符串b任意，只要不相交即可。
  • 如果字符串a右段包含k个连续的‘G’，那么字符串b的左端包含N-k个连续的‘G’或者b中含有N个连续‘G’。
  • 否则a的右段无‘G’，那么要求b要包含N个连续的‘G’。（可以归到第二点中）。
    所以，其实就是对字符串建立个数索引，情况1简单，情况2就是对b进行建立索引，建立索引的是O(N ^ 3)的，可以从母字符右端 - 1开始向左枚举a的右端点，这样可以增量地计算出b的索引。
